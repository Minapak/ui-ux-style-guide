07 UI Design Style

Brutalism

Raw, unpolished web — default fonts, harsh borders and stark contrast that break the rules on purpose.

Emerged: resurging since ~2016Niche · trendingStrong in creative & indie

👁At a glance

Web brutalism, borrowed from architecture, is an anti-design style that exposes raw HTML elements, system fonts, harsh borders and blunt structure instead of polishing them. It weaponizes ‘crudeness’ to convey strong attitude, authenticity and rebellious personality.

🧬Key characteristics

Raw structure

Default HTML and system/monospace fonts left exposed

Harsh borders

Thick black lines and hard offset shadows

Stark contrast

Mostly black/white with primary accents; no rounding

Deliberate defiance

Grid/alignment rules broken on purpose for tension

⚖️Strengths vs Limitations

Presented neutrally. Weigh them against your project's goals.

✅ Strengths

  • Bold and original — highly memorable
  • Unpolished look reads as authentic and full of personality
  • Simple CSS — fast and lightweight to build
  • High contrast can actually aid readability
  • Feels fresh to users tired of slick trends

⚠️ Limitations

  • Crudeness can undermine trust/professionalism
  • Breaking conventions can hurt usability and learnability
  • Polarizing taste — poor fit for mass-market services
  • Thin line between ‘intentional’ and ‘just badly made’
  • Hard to keep consistent as it scales/maintains

🎯When it fits / when to avoid

👍 Good fit when

  • Portfolio, agency and artist sites
  • Events, exhibitions, cultural projects
  • Rebellious / indie brand identities
  • Experimental campaign landings

👎 Avoid when

  • Finance, healthcare, commerce where trust is core
  • Products for large, age-diverse audiences
  • Productivity tools with complex workflows
  • Contexts demanding strict accessibility/consistency

🧾Outsourcing brief (Upwork)

Brutalism — Outsourcing brief (Upwork)

  • Communicate ‘intentional defiance’ via mockups — document rules so devs don’t treat it as a bug
  • Uses system/monospace fonts and native elements — low web-font overhead
  • Fix hard-shadow/thick-border values as tokens to keep ‘consistent crudeness’
  • Accessibility: require focus rings and keyboard nav to survive the chaos
  • Agree breakpoint rules so the ‘broken’ grid doesn’t actually break responsively
